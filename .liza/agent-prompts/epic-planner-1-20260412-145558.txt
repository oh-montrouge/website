EPIC-WRITING SKILL:
Invoke skill: epic-writing when writing each epic for this task.

Key principles from the skill:
- An epic bounds one cohesive capability area, expected to decompose into 3–8 user stories
- Each capability section is a Story Writer task brief — must be self-contained
- The Story Writer reads Personas + General Information + capability section — nothing else
- Completion criteria must be falsifiable: all story ACs passing satisfies them
- Personas must include environment/skill level where it affects product behavior
- Out of Scope is not optional — it protects Story Writers from scope absorption
- Assumptions resolve behavioral ambiguity only — no premature solutioning
- If a single epic spans multiple independent capability areas, it is too broad — flag for splitting
- Producing multiple well-sized epics from one vision source is normal planner behavior

Anti-patterns to avoid:
- Vision Laundering: capabilities named after technical layers instead of user-facing behavior
- Giant Epic: multiple independent persona clusters or unrelated subsystems in one epic
- Dwarf Epic: single user action with one AC — this is a story, not an epic
- Storified Epic: capabilities written at story granularity (Given/When/Then)
- Unfalsifiable Completion Criteria: "users have a better experience" — not falsifiable
- Scope Absorption: vision says X, you write capabilities for X, Y, and Z
- Opaque Capability: Story Writer needs to re-read the vision to understand what to build
